Get Pass by Correlation ID

When an Organization creates a Pass for a Customer using PassWick they must also providea correlation ID. This correlation ID is an identifier defined solely by the Organizationand should be an internal ID that is unique to the Organization's Customer. The correlation ID can be used to quickly retrieve passes specific to their Customer. This endpoint returns the binary representation of a pass that can be installed directly into the Apple Wallet.

Delete Pass

When an Organization creates a Pass for a Customer using PassWick they must also providea correlation ID. This correlation ID is an identifier defined solely by the Organizationand should be an internal ID that is unique to the Organization's Customer. The correlation ID can be used to quickly retrieve passes specific to their Customer. This endpoint deletes a Customer's pass by correlation ID. Deleting a Pass does not removeit from the Customer's wallet.

A Shareable Link to a Pass

Each Organization Customer Pass has a unique serial number generated by PassWick. This serial number is returned when a Pass is created for the Customer. This serial number can be used to quickly retrieve passes. This endpoint represents a shareable link that can be sent to Customers for them to click and quickly install pass in their Apple Wallet.

path Parameters
orgAuthId
required
string

Your Organization's authorization ID - available on PassWick Admin console

passTypeIdentifier
required
string

Your Organization's pass type identifier - available on PassWick Admin console

serialNumber
required
string

Pass Serial Number generated by PassWick and available in your Customer pass list

Responses

https://localhost:8081/api/pws/v1/ng/pass/customer/pkpass/{orgAuthId}/{passTypeIdentifier}/{serialNumber}

Each Organization Customer Pass has a unique serial number generated by PassWick. This serial number is returned when a Pass is created for the Customer. This serial number can be used to quickly retrieve passes. This endpoint deletes a Customer pass by the Pass serial number. Deleting a Pass does not remove it from the Customer's wallet.

Touch Pass

When an Organization creates a Pass for a Customer using PassWick they must also providea correlation ID. This correlation ID is an identifier defined solely by the Organizationand should be an internal ID that is unique to the Organization's Customer. The correlation ID can be used to quickly retrieve passes specific to their Customer. This endpoint will change the pass last updated date and refresh the pass by pulling external data (if so configured). This will result in an update to the Customer's pass in their wallet.

correlationId
required
string

correlationId

Responses
